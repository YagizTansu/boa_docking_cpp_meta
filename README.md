# BOA Docking C++ Meta Workspace

This repository contains a ROS 1 workspace dedicated to automated pallet detection and precision docking operations. It uses 2D LIDAR data for perception and a PID-based path tracking algorithm to navigate the robot to the pallet accurately.

## Packages

The workspace consists of two main ROS packages:

### 1. `pallet_detection_and_docking`
This is the core custom package developed for this project. It is responsible for the perception and high-level control of the docking sequence.
- **Pallet Detection (`pallet_detection.cpp`)**: 
  - Subscribes to the `/scan_fork` LIDAR topic.
  - Filters the laser scan points using a Region of Interest (ROI) polygon to ignore irrelevant environmental data.
  - Converts the filtered scans to PCL point clouds and performs DBSCAN clustering to identify distinct objects.
  - Computes the convex hull and the minimum area bounding rectangle for each cluster using OpenCV.
  - Calculates the exact center position and orientation (yaw) of the pallet's front face, publishing the calculated pose as visual markers (`visualization_msgs::Marker`) for RViz.
  
- **Pallet Docking (`pallet_docking.cpp`)**: 
  - Acts as the docking orchestrator. 
  - Subscribes to `pallet_station_pose` and tracks the robot's current pose (`/tf_pose`).
  - Calculates a precise path traversing from the robot to the target pallet pose.
  - Handles the pickup (`executePickPallet`) and dropoff logic via ROS services (`/Services`), sending pause/resume signals and managing linear motor activations.
  - Generates the path for the `tracking_pid` controller and monitors the `follow_path/status` to determine when docking is successfully completed.

### 2. `tracking_pid`
An integrated third-party ROS package that provides a robust PID controller for following a moving target (carrot). In this project, it is used to track the path generated by the `pallet_docking` node, ensuring the robot smoothly aligns with the pallet during the final approach phase without oscillations.

## How the Algorithms Work (Simplified)

### 1. Pallet Detection Algorithm
* **Filtering:** The robot's LIDAR scanner sees thousands of laser points. To save processing time, we define a small invisible box in front of the robot (Region of Interest) and delete any points outside of it.
* **Clustering (DBSCAN):** We group the remaining points based on how close they are to one another. Points that clump together form a distinct physical object, like our pallet.
* **Shape Extraction:** For each group of points, the code draws a tight rectangular box around them. Pallets have straight front edges, so finding this box helps us identify the pallet's front face.
* **Pose Calculation:** By finding the center of that rectangular box's edge, the algorithm figures out exactly where the pallet is (its X, Y coordinates) and which direction it is facing (its Yaw angle).

### 2. Docking & Path Tracking Algorithm
* **Path Generation:** Once the pallet's exact position is known, the robot plots a smooth line (path) from where it currently is to the pallet.
* **Path Tracking (PID Controller):** The robot acts like a car trying to stay precisely in the middle of a lane. If the robot drifts slightly off the path, the PID controller immediately calculates this "error" and steers the wheels back on track. It continuously adjusts the speed and steering to make a smooth approach.
* **Sequence Control:** The system acts as a smart manager. It waits until the robot perfectly reaches the end of the path along the pallet. Once confirmed, it automatically triggers the robot's hardware (like linear motors) to pick up or drop off the pallet.

## How to Run

To launch the full docking stack (including the PID controller and pallet detection nodes), you can use the provided launch file:

```bash
roslaunch pallet_detection_and_docking boa_docking.launch
```

*This will start the `path_interpolator`, `controller` (from tracking_pid), `pallet_detection`, and `pallet_docking` nodes.*
